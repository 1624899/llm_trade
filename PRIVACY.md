# 隐私政策草案

本政策说明 LLM-TRADE 社区版可能如何处理用户信息。若后续提供在线服务，应结合实际部署方式、服务器所在地和服务对象进行合规复核。

## 1. 信息处理范围

本地单机版默认主要在用户设备内处理数据，可能包括：

- 用户配置和 API Key。
- 本地行情、财务、新闻缓存和报告。
- 观察仓、模拟仓、交易流水和复盘记录。
- 运行日志、错误日志和任务日志。

用户主动提交 Issue、讨论或问题反馈时，可能额外提供：

- 软件版本、操作系统、错误日志和配置摘要。
- 服务沟通记录、问题复现步骤和必要截图。
- 必要的使用统计、错误摘要和安全审计信息。

## 2. 处理目的

信息处理用于提供软件功能、排查故障、改进产品稳定性和履行用户协议。

## 3. 最小必要原则

产品应尽量减少上传信息。除错误诊断或用户主动提交外，观察仓、模拟仓、报告内容和 API Key 应优先保存在本地。

## 4. 第三方服务

用户配置的大模型、搜索、行情和财务数据源可能由第三方提供。用户使用这些服务时，应同时遵守第三方隐私政策和服务条款。产品应避免把不必要的个人信息、真实持仓或敏感配置发送给第三方模型。

## 5. 安全措施

产品应对 API Key、授权文件和日志中的敏感字段进行脱敏或加密存储。授权服务器应记录必要审计日志，并限制内部访问权限。

## 6. 用户权利

用户可以请求查看、更正、导出或删除其主动提交的问题反馈信息。对于本地单机数据，用户可通过删除本地数据库、配置文件和输出文件完成清理。

## 7. 数据保留

问题反馈和审计信息应仅在排查问题、安全审计或法律要求所需期间保存。超过必要期限后应删除或匿名化处理。
