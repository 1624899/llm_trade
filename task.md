# LLM-TRADE 演进任务清单

## 4. 图表多模态分析（长期目标）

### 建议方向

条件允许时，将 K 线图、成交量、MACD、筹码分布或支撑压力图生成截图，交给支持视觉能力的模型分析。

优先级低于财务数据和退出机制，因为多模态依赖模型能力和图表渲染稳定性，工程复杂度更高。

## 5. 交易 Agent 与真实持仓生命周期模拟（已落地第一版 / 待实盘前继续打磨）

### 当前目标

把系统从“推荐股票 + 观察仓复盘”推进到“观察仓候选池 + 交易仓模拟 + 亏损反思闭环”。

新增独立 `--trade` 命令，交易 Agent 拥有一个持久模拟账户，初始资金 10000 元。账户只初始化一次，后续现金、持仓、交易流水和盈亏持续滚动，不在每次运行时重置。

### 设计约束

- 观察仓最多维护 10 只股票及其最新分析。
- 交易仓最多同时持有 5 只股票。
- 每次运行时的当前行情快照作为模拟成交价格。
- 买入后至少持有 5 个交易日。
- 卖出后 5 个交易日内不允许回买同一股票。
- 禁止频繁交易，默认单次运行最多 2 笔买入、2 笔卖出。
- 基本面、技术面或资讯风险明显恶化时，允许提前减仓或清仓。
- 系统仍然只做研究和模拟，不接入真实券商，不做实盘下单。

### 实施计划

- [x] 数据库 schema 与兼容迁移
  - 新增 `watchlist_items` 表，用于替代 `paper_trades` 的观察仓职责。
  - 新增 `trading_account` 表，保存初始资金、现金、总市值、总权益、已实现盈亏。
  - 新增 `trading_positions` 表，保存交易仓持仓、成本、数量、现价和浮盈亏。
  - 新增 `trade_orders` 表，保存 BUY/SELL/HOLD/WATCH/REMOVE 动作和交易理由。
  - 保留旧 `paper_trades`，避免历史测试和旧数据直接失效。

- [x] 观察仓重构
  - 新增 `src/evaluation/watchlist.py`。
  - `--pick` 只更新观察仓，不直接等同买入。
  - 观察仓条目保存推荐分层、预期收益、推荐理由、基本面分析、技术面分析、资讯风控和当前价格。
  - 新推荐进入时，优先保留强推荐和配置/轻仓验证标的，风险恶化或质量较低标的可降级或移除。

- [x] 交易账户与交易流水
  - 新增 `src/evaluation/trading_account.py`。
  - 首次运行自动创建默认账户并注入 10000 元。
  - 后续运行只读取并更新现有账户，不重置资金。
  - 买入按 A 股 100 股一手校验现金。
  - 执行成交后同步更新现金、持仓、浮盈亏和交易流水。

- [x] TradingAgent 决策模块
  - 新增 `src/agent/trading_agent.py`。
  - 输入观察仓、交易仓、现金、当前价格、宏观环境、ExitAgent 信号和历史反思规则。
  - 输出结构化交易决策：买入、卖出、持有、继续观察或移出观察仓。
  - 对低风险且中期预期收益大于 15% 的股票给出清晰配置判断。
  - 对基本面恶化、技术破位、放量滞涨、趋势背离或资讯硬风险标的明确减仓、清仓或回避。
  - LLM 失败时使用规则兜底，避免生成乱单。

- [x] CLI 新增 `--trade`
  - 在 `main.py` 新增 `--trade` 参数。
  - `AgentCoordinator` 新增交易流程入口。
  - 交易流程顺序：刷新价格 -> 获取宏观环境 -> 评估退出信号 -> 调用 TradingAgent -> 校验交易纪律 -> 写入交易流水 -> 输出交易报告。

- [x] 亏损反思升级
  - ReflectionAgent 不再只基于观察仓浮亏反思。
  - 反思输入增加推荐内容、买入理由、卖出理由、持有天数、交易动作和最终盈亏。
  - 反思规则继续沉淀到 `data/rules_book.txt`，供 DecisionAgent 和 TradingAgent 共同加载。

- [x] 测试与回归验证
  - 覆盖新表自动创建和旧数据兼容。
  - 覆盖观察仓最多 10 只。
  - 覆盖交易仓最多 5 只。
  - 覆盖账户初始资金只初始化一次。
  - 覆盖买入、卖出、持有流水写入。
  - 覆盖最短持有期和卖出冷却期。
  - 覆盖风险恶化时提前清仓。
  - 回归 `--pick`、`--post`、ExitAgent、ReflectionAgent 现有测试。
